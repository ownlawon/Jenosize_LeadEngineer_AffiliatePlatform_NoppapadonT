# Accessibility Notes

This page records the accessibility hooks that exist in the codebase and how
to verify them.

## How to audit

Two complementary tools work well:

```bash
# axe-core CLI — best for catching ARIA/contrast/landmark issues
npx @axe-core/cli https://jenosizeweb-production.up.railway.app/c/<id>

# Lighthouse — adds tab-order + keyboard analysis, included with Chrome
npx lighthouse https://jenosizeweb-production.up.railway.app/c/<id> \
  --only-categories=accessibility --view
```

Run both against the **deployed Railway URL** for accurate results. Dev mode
inserts React DevTools markers and noisy warnings that aren't in production.

## What's already in place

| Affordance                                           | Where                                   | Notes                                                                        |
| ---------------------------------------------------- | --------------------------------------- | ---------------------------------------------------------------------------- |
| Skip-to-main-content link                            | `app/layout.tsx`                        | Visually hidden, focus-revealed; targets `#main`                             |
| `<main id="main">` landmark                          | All public + admin pages                | Skip link target; one per page                                               |
| `<nav aria-label="Primary">`                         | `components/Nav.tsx`                    | Disambiguates from in-content nav                                            |
| Logo `aria-label`                                    | `components/Nav.tsx`                    | Image-only link gets a name                                                  |
| `aria-current="page"` on active nav item             | `components/Nav.tsx`                    | Screen reader announces current location                                     |
| `role="status" aria-live="polite"` on Skeleton       | `components/Skeleton.tsx`               | Loading state announced, not silent                                          |
| `aria-describedby` for Reset hint                    | `components/ResetDemoButton.tsx`        | Tooltip linked to button                                                     |
| `role="radiogroup" / role="radio"` segmented control | `app/admin/products/AddProductForm.tsx` | Marketplace selector behaves like a radio group                              |
| `aria-hidden` for decorative icons (←, →, monogram)  | Multiple files                          | Excluded from a11y tree                                                      |
| Sonner toasts                                        | `app/layout.tsx`                        | Library default uses `aria-live="polite"` for success, `assertive` for error |

## Keyboard navigation

The full Add → Generate → Click flow is reachable with Tab + Enter only:

1. Tab through nav → land on Products
2. Tab into URL input, type, Tab to Submit, Enter
3. Tab into Campaigns nav, repeat for campaign create
4. Tab into Links nav, Tab through dropdowns, Tab to Generate
5. Tab to short-link cell → Enter follows the drill-down link
6. Tab to "Open redirect" → Enter follows the affiliate redirect

No keyboard traps observed; focus rings preserved (Tailwind's defaults are
visible against the slate palette).

## Screen reader spot checks

- VoiceOver on macOS reads the public landing as a single document with one
  banner, one main, and a list of campaign cards with accessible names from
  their headings.
- Toasts on admin actions are announced once, not duplicated.
- Loading skeletons announce "Loading" rather than going silent.

## Known limitations

- **Recharts SVG charts** lack accessible alternatives. For an analytics-
  reviewer audience the chart is supplementary to the headline numbers shown
  in the KPI cards above, so the impact is bounded — but a future pass could
  add `<title>`/`<desc>` to the chart SVG and a `aria-describedby` summary.
- **Color-only signals**: the green "Best price" badge uses both colour AND
  the text "Best price" so it's not purely colour-based, but a colour-blind
  user perceiving green and red the same will still read the textual cue.
